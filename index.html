// the problem

Game servers are soft targets.
Generic DDoS tools don't fix that.

Volumetric floods aim at your IP. Application-layer attacks exploit your game protocol. Most protection sits in front of your server, adding latency, introducing proxy failure points, and filtering traffic blind to what FiveM or Minecraft actually looks like.

// 01

Protocol-blind filtering

Generic L4 tools can't distinguish a legitimate FiveM handshake from a crafted flood packet. Attackers exploit this. You get false drops or no effective protection.

// 02

Proxy latency tax

Traffic-scrubbing proxies add 10–40ms of round-trip overhead. For game servers where 20ms is the difference between playable and broken, that cost is not acceptable.

// 03

Coarse rate limiting

IP-level thresholds that ignore game session state punish legitimate players during attack spikes. You lose players you should have kept connected.

// how it works

Filtering at the earliest possible hook

Kalkan loads eBPF programs into the kernel per customer configuration. Malicious traffic is absorbed before the OS networking stack processes it. The game process never sees it.

// supported platforms

Built for the games being attacked

FiveM, Minecraft, and Source engine servers face attacks that generic tools were never designed to handle. Kalkan knows the difference.

// FiveM

FiveM

FiveM servers are among the most actively targeted in game hosting. Most operators run on infrastructure with no protocol-aware protection. Attackers know this and exploit it with high-volume UDP floods and fake session spam that exhaust server resources before a single real player connects.

With Kalkan, attack traffic is identified and dropped in the kernel before FXServer processes it. Your server stays online and your player count stays intact.

// Minecraft

Minecraft

Minecraft is the highest-volume DDoS target in game hosting. Login bot floods, TCP connection storms, and disconnect-reconnect loops are daily occurrences for any server with real traffic. BungeeCord and Velocity setups add additional attack surface.

Kalkan stops bot floods and connection storms before they reach your server. Legitimate players connect normally. BungeeCord and Velocity are fully supported.

// Source Engine

Source Engine

A2S query reflection is the defining attack vector for Source engine servers. A single crafted query triggers a response large enough to saturate your uplink. The game itself never gets touched, but it goes offline anyway. CS2, Rust, and Garry's Mod operators all face this.

Kalkan rate-limits query traffic at the kernel level before your bandwidth is consumed. Game ports stay open. No separate configuration per game.

// differentiators

Not another proxy in front of your server

ebpf_kernel_space

eBPF runs in the kernel, not in front of it

Most DDoS protection routes your traffic through an external scrubbing proxy. That adds latency, creates a new failure point, and means your traffic is handled by infrastructure you don't control. Kalkan uses eBPF to filter directly in the Linux kernel on your own server. No detour, no proxy overhead.

protocol_aware

It knows what your game traffic looks like

Generic tools see packets. Kalkan sees FiveM connection requests, Minecraft login sequences, and Source engine queries. That distinction is what makes it possible to stop attack traffic without dropping real players. It cannot be replicated with a tool that doesn't understand your protocol.

modular_programs

No shared filter chain

Your server loads only the protection it needs. A FiveM server runs FiveM-specific eBPF programs. A Minecraft server runs Minecraft-specific programs. There is no generic ruleset applied to everyone. Less false positives, less overhead, and protection calibrated to your actual traffic.

real_time_visibility

You see what's happening in real time

Every attack event, every blocked IP, every spike in drop rate is visible in a live dashboard with alerts sent to wherever you want them. You are not flying blind during an attack. You know when it started, how large it is, and when it stopped.
